from sklearn.cluster import AgglomerativeClustering
from sklearn.metrics import silhouette_score

SpendingAnnual = []

for i in range(2,11):
    SpendingAnnual.append(silhouette_score(df, AgglomerativeClustering(n_clusters = i).fit_predict(df)))
    
plt.bar(x=range(2,11), height=SpendingAnnual)
plt.xlabel('Number of Clusters')
plt.ylabel('Spending Score and Annual Income')
plt.title("Scores")
plt.show()

#STEP 5

#In K-means clustering, it use a prespecifice number of clusters, the method assigns record to each cluster to find the 
#mutually exclusive cluster of spherical shape based on distace. Kmeans clustering needed advance knowledge of K number of 
#cluster one want to divide the data.

#In Hierarchical clustering, the methods can be either divisive or agglomerative. In hierarchical clustering, one can stop
#at any number of clusters. Hierarchical clustering requires the computation and storage of an nxn distance matrix. For 
#very large datasets, this can be expensive and slow. 

#The advantages of using Kmeans is covergence is guaranteed. Specialized to clusters of different sizes and shape. 
#The disadvatage of using kmeans is K-value is difficult to predict. Didn't work well with global cluster

#The advantage of using Hierarchy clustering is ease of handling of any forms of similarity or distance. Consequently,
#applicability to any attributes type. The disadvantage is computation cost.

#For me the hierarchy clustering take more time then the kmeans clustering


#I would use KMeans if there is a specific number of clusters in the dataset, but the group they belong to is unknown. Also,
#any data with a large number of variables because the computation time of hierarchical is expensive

#I would use Hierarchical clustering if the number of variables is low, if the distinguishes are based on prior beliefts,
# hierarchical clustering should be used to know the number of cluster because it is easier to determine the number of
#cluster with the dendrogram
